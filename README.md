# 日本語 N3 ことばゲーム

中国語母語話者向けの JLPT N3 語彙学習ゲームです。`words.js` の語彙データを読み込み、**かな→中国語訳** の4択問題を中心に出題します。

## 特徴

- 出題データは `words.js` をそのまま利用
- 出題文はかな表示（例: 「よてい」のいみはどれ？）
- 選択肢は通常モード・聞き取りモードともに中国語のみ
- 正解／不正解表示では `jp`（漢字かな交じり）を確認可能
- `trap` がある語は、正解後に「メモ」として表示
- 品詞（`pos`）が偏りすぎないようにラウンドロビンで出題
- 場面カテゴリ（`topic`）も各品詞内で偏りすぎないように調整
- 同一問題内の選択肢重複を防止
- 複数問題間の誤答選択肢・選択肢セット重複を抑制
- 1プレイ最大100問（語彙数が100未満なら語彙数ぶんだけ安全に出題）

## ファイル構成

- `index.html`: 画面構造。`./words.js` → `./app.js` の順で読み込み
- `style.css`: レイアウト・配色
- `words.js`: 語彙データ（`jp` / `kana` / `zh` / `pos` / `topic` / `trap`）
- `app.js`: 出題制御、選択肢生成、重複抑制、スコア管理

## 使い方

1. ローカルでサーバーを起動

   ```bash
   python3 -m http.server 8000
   ```

2. ブラウザで `http://localhost:8000` を開く
3. 「次の問題」で進める（聞き取り問題では「もう一度聞く」が使用可能）

## `words.js` の読み込み方式

GitHub Pages でも動くよう、相対パスで読み込みます。

```html
<script src="./words.js"></script>
<script src="./app.js"></script>
```

`words.js` 側で定義した `words` 配列を、`app.js` が直接参照して問題を生成します。

## 選択肢重複を減らす仕組み

`app.js` では次の状態を保持して、短い間隔で同じ誤答が出続けないようにしています。

- `usedDistractors`: 直近の誤答選択肢（中国語訳）履歴
- `recentChoices`: 直近で画面に出た選択肢全体の履歴
- `usedChoiceSets`: 直近で使った4択セット（ソート済みキー）

優先ロジック:

1. 直近履歴にない誤答候補を優先
2. 1問内では `Set` 相当の処理で選択肢重複を排除
3. 4択セットが最近と同じになった場合は再シャッフルして回避
4. 候補不足時のみフォールバック候補を補充（安全動作）

## データ形式（例）

```js
{
  jp: "予定",
  kana: "よてい",
  zh: "计划；预定",
  pos: "名詞",
  topic: "時間",
  trap: "予約・約束との使い分けに注意"
}
```
